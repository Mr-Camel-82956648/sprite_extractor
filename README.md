# Sprite Extractor

将 MP4 视频转换为 HTML 游戏可用的动作序列帧（透明背景 PNG）。提供可视化 Web 工具完成抽帧、帧选择、循环预览、命名导出、抠图导入、多动作尺寸统一的全流程。

## 快速开始

```bash
cd sprite_extractor

# 首次使用：创建虚拟环境
python -m venv venv
venv\Scripts\pip install flask opencv-python

# 启动
venv\Scripts\python server.py
```

浏览器打开 http://localhost:5000

## 使用流程

页面打开后只显示步骤 1 和步骤 6。步骤 2-5 会在点击"抽帧"后自动出现。

### 单个视频处理（步骤 1-5，需对每个视频重复）

1. **将视频放入 `source_videos/`**（支持 mp4/avi/mov/webm）
2. **步骤 1 — 选择视频 & 抽帧**：从下拉框选视频，设置抽帧频率（每 N 帧取 1 帧），点击"抽帧"
3. **步骤 2 — 选择帧**：以缩略图展示所有抽出的帧，点击选择/取消
   - Shift+点击：范围选择
   - 去掉开头停顿帧和中间重复帧
4. **步骤 3 — 循环预览**：选中的帧按顺序循环播放，拖动 FPS 滑块调节速度，确认动画连贯
5. **步骤 4 — 命名 & 导出**：填写角色名、方向、动作，点击"下载 ZIP"
   - 命名格式：`{角色}_{方向}_{动作}_{序号}.png`
   - 示例：`player1_front_right_run_01.png` ~ `player1_front_right_run_06.png`
6. **步骤 5 — 导入抠图结果**：将 ZIP 解压后的图片上传到抠图网站（如 koukoutu.com）批量去背景，然后将抠图后的 PNG 拖入页面导入

> **处理多个视频**：完成一个视频的步骤 1-5 后，回到步骤 1 选下一个视频继续。每次点"抽帧"步骤 2-5 会刷新。修改步骤 4 的方向/动作字段即可为不同动作生成不同命名。

### 多动作尺寸统一（步骤 6，所有视频处理完后执行一次）

同一角色的不同动作视频可能素材大小不一致，步骤 6 用于统一：

1. 点击"刷新文件夹列表"，列出 `output/` 下所有已导入的动作文件夹
2. 勾选需要统一的文件夹（通常是同一角色的所有动作）
3. 每个文件夹可选一张代表帧，在预览区并排对比
4. 分别调整每个文件夹的**缩放**、**X 偏移**、**Y 偏移**滑块，直到角色大小和位置一致
5. 拖动**画布尺寸**滑块设定输出尺寸（如 960 或 720）
6. 点击"应用缩放并导出"，所有帧按设定处理，结果保存在 `output/{文件夹名}_normalized/`

## 目录结构

```
sprite_extractor/
├── source_videos/   # 原始视频（手动放入）
├── frames/          # 抽帧中间产物（自动生成，可删除）
├── output/          # 抠图后导入的序列帧 & 归一化结果
├── server.py        # Flask 后端
├── index.html       # 前端工具页面
└── venv/            # Python 虚拟环境（不提交）
```

## 依赖

- Python 3.10+
- Flask
- OpenCV (opencv-python)
- NumPy（随 OpenCV 自动安装）
